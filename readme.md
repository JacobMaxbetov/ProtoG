# ProtoG

**ProtoG** — платформа по продаже курсов с интерактивными уроками и системой оплаты.

## 🚀 Быстрый старт

### Требования
- **Node.js** (версия 16+)
- **Python** (версия 3.8+)
- **npm**

### Установка зависимостей

## 1. Установка Node.js зависимостей

```bash
npm install
```

## 2. Установка зависимостей бэкенда (Node.js)

```bash
cd backend && npm install
cd ..
```

## 3. Установка Python зависимостей (важно!)
```bash
cd backend
pip install flask flask-cors python-dotenv PyJWT
cd ..
```

```bash
# Установка зависимостей корневого проекта
npm install

# Установка зависимостей бэкенда
cd backend && npm install

# Установка зависимостей фронтенда
cd ../frontend && npm install
```

## 🏃‍♂️ Запуск приложения

### Запуск отдельных компонентов

```bash
# Запуск фронтенда (порт 3000)
npm run client

# Запуск бэкенда API (порт 5000)
npm run server

# Запуск сервера уроков (порт 5551)
python backend/app.py
```

## 🛠 Технический стек

### Фронтенд
- **Vue.js 3** - Фреймворк для пользовательского интерфейса
- **Vite** - Сборщик и dev-сервер
- **Axios** - HTTP клиент

### Бэкенд
- **Node.js + Express** - API сервер
- **Python + Flask** - Сервер уроков
- **SQLite** - База данных
- **JWT** - Аутентификация
- **Socket.IO** - WebSocket соединения

### Платежи
- **YooKassa** - Система оплаты

## 📜 Доступные скрипты

### Корневой уровень
- `npm run dev` - Запуск фронтенда + бэкенда одновременно
- `npm run client` - Запуск только фронтенда
- `npm run server` - Запуск только бэкенда API
- `npm run content` - Запуск сервера контента
- `npm run build` - Сборка фронтенда для продакшена
- `npm run preview` - Предпросмотр собранного фронтенда

### Бэкенд
- `npm run dev` - Запуск с nodemon (автоперезагрузка)
- `npm run start` - Запуск продакшен версии

### Фронтенд
- `npm run dev` - Запуск dev-сервера Vite
- `npm run build` - Сборка для продакшена
- `npm run preview` - Предпросмотр сборки

## 🌐 Доступ к приложению

После запуска всех компонентов:

- **Фронтенд**: http://localhost:3000
- **Бэкенд API**: http://localhost:5000
- **Сервер уроков**: http://localhost:5551

## 🔧 Настройка

1. Создайте файл `.env` в папке `backend/` с настройками:
   ```
   PORT=5000 -- не трогайте во избежания ошибок
   JWT_SECRET=secret

   NODE_ENV=development


   YOOKASSA_SHOP_ID=key
   YOOKASSA_SECRET_KEY=key

   TEACHERS=usernames

   ADMINS=usernames
   ```

2. Инициализируйте базу данных:
   ```bash
   cd backend
   node scripts/init-db.js
   ```

## 📝 Примечания

- Сервер уроков (`python backend/app.py`) отвечает за отображение интерактивных уроков
- Бэкенд API (`npm run server`) предоставляет REST API для управления пользователями, курсами и платежами
- Фронтенд (`npm run client`) - пользовательский интерфейс на Vue.js

## 🤝 Поддержка

При возникновении проблем проверьте:
1. Установлены ли все зависимости (`npm install` в каждой папке)
2. Запущены ли все необходимые сервисы
3. Правильно ли настроены переменные окружения


## 🌍 Доступ по доменному имени

Для доступа к приложению по доменному имени используйте переменные окружения с префиксом `VITE_*` в файлах `.env`.

### Текущий статус
Проект успешно запущен и протестирован. База данных и контент уроков доступны через туннель **CloudPub.ru**. 

### Настройка туннелей CloudPub (для продакшена / тестирования)

1. Скачайте и установите клиент CloudPub для вашей ОС с [официального сайта](https://cloudpub.ru).

2. Привяжите приложение к аккаунту:
   ```bash
   clo.exe set token ВАШ_ТОКЕН
3. Опубликуйте локальные порты в отдельных терминалах:
```bash
clo.exe publish http 3000     # Фронтенд
```
```bash
clo.exe publish http 5000     # Основной API
```
```bash
clo.exe publish http 5551     # Сервер уроков (Python)
```

4. После получения доменов от CloudPub обновите переменные VITE_* в файле .env (в корне проекта и backend, frontend).
